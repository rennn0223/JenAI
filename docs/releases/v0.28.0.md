# JenAI v0.28.0 - Safety boundaries and reliable runtime inputs

**Fail closed at the edges: stale approvals, malformed numeric values, missing twin telemetry, and generated package metadata can no longer silently cross a safety boundary.**

## Safety

- WebUI confirmation tokens now expire after 120 seconds, remain one-time, and are all revoked by STOP so an old confirmation cannot restart motion after an emergency stop.
- `/api/stop` executes without waiting for a request body; other JSON endpoints reject malformed lengths and bodies larger than 64 KiB.
- Vehicle commands and pose/forbidden-zone schemas reject NaN and infinity, while low-level velocity clamping fails closed for invalid values or limits.
- Twin Gate G3 now returns `refer` when forbidden zones are configured but no twin pose sample was observed, instead of treating missing telemetry as proof that the route stayed clear.

## Reliability

- Configuration writes use a unique same-directory temporary file, flush and `fsync` before atomic replacement, and clean up failed writes.
- Generated ROS2 package descriptions are escaped correctly for both `package.xml` and `setup.py`.
- Standalone greetings in the TUI use a tool-free streaming chat path, avoiding unnecessary specialist handoffs while preserving normal robot routing for action requests.
