# JenAI v0.32.1 - Timeout 清理路徑補洞

**v0.32.0 引入的 process-group timeout 清理有兩個縫:逃出 session 的子孫行程能讓 shell tool 永久卡死、非 timeout 的例外會洩漏子行程。本版把每一條退出路徑都堵上。**

## 修正(對 v0.32.0 的 code review 結果)

- **timeout 清理不再可能永久阻塞**:原本 group SIGKILL 之後用無上限的 `communicate()` 等輸出;若子孫行程以 `setsid`/daemonize 自立 session 並持有 stdout/stderr pipe,SIGKILL 殺不到它,`run_shell` 的執行緒會永久卡住。現在 timeout 路徑每一步等待都有上限(TERM → 1s → KILL → 1s → 關閉 pipe、只 reap 直接子行程),保證回報 exit 124
- **任何例外都會清理子行程**:`run_process` 改以 `except BaseException` 統一終止整個 process group(對齊 `subprocess.run` 的行為),Ctrl-C 或取消不再留下還在跑的 colcon/shell 與洩漏的 pipe
- **`_run_process` 改名公開 `run_process`**:它是 `shell_core` 供 `ros2_pkg_core` 使用的跨模組 API,不再以底線佯裝模組私有;`build_package` 殘留的 function-local `import os` 一併移到模組頂部

## 已記錄、待另行處理

- Emergency stop 蓋不掉進行中的 `ros_drive`(halt 的零速脈衝會被 drive 的 10Hz 發布流覆蓋,最長 30 秒)——既有的安全鏈設計缺口,非 v0.32.0 引入;修法需讓 stop 能終止 drive publisher 行程,屬設計變更且需實機驗測,詳見 PR #74 說明
