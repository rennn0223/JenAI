# JenAI v2.0.0

> **歷史未發布候選說明：** `v2.0.0` annotated tag 已建立，但該次 workflow 在
> user-owned private repository 的 GitHub provenance step 失敗，因此從未建立 GitHub
> Release，也沒有發布任何 assets。以下保留當時候選版的預期行為與本機驗證，不是已發布
> 證據；發行工作流修正見 [v2.0.1](v2.0.1.md)。

**v2.0.0 是安全語意與產品化生命週期的 major release。它仍是受監督、具執行邊界的
ROS2 高階決策與工作流代理；本版沒有實作或宣稱常駐 M6 自主決策迴圈。**

## Breaking safety migration

- P2 與所有 `HOST_COMMAND` 不再接受自動批准或 session remember；每次執行都必須重新
  顯示批准卡，且只提供明確的 **Yes／No**，P2 預設選取 No。
- 舊 session 中曾記住的 P2／host 批准不再生效。這是刻意的 major-version 安全收緊，
  管理者不可用設定或自動模式恢復舊行為。
- 有界、非 host 的 P0／P1 操作仍可依政策使用 session remember／auto；`/stop` 仍免批准。

## Cancellation and hard stop

- shell、ROS2 one-shot publish 與 bounded publisher 改走可取消的 async process-group
  lifecycle；timeout／取消先 TERM、等待 reap，必要時再 KILL，避免遺留子程序。
- `/stop` 採兩階段順序：先取消並等待前一個致動程序清理，再執行 bridge halt／零速度；
  速度型 ROS publish 取消時亦送保守的 final zero，避免「先停、後補發舊動作」。

## Local data lifecycle

- 新增 `JenAI data status|harden|export|prune|purge`，涵蓋 sessions、pending runs、
  locations、reports、traces、audit 與 config backups。
- 新檔案使用 private mode 與 atomic replace；legacy 權限修復必須先檢視 plan 再明確確認，
  export 使用 allow-list 並排除 config／credentials。詳細語意見
  [DATA_LIFECYCLE](../DATA_LIFECYCLE.md)。

## TUI and operator workflow

- 保留目前 Claude Code 風格整體設計與既有會動的臘腸狗；welcome metadata 精簡為
  model/provider/config，不再把 locations／pass 行塞在首頁。
- command palette 與 welcome/composer/statusline 會依終端寬高調整；120×30、80×30、
  50×30 與 50×20 viewport 皆有回歸測試。這是 responsive 修正，不是未經批准的視覺改版。

## Isaac HIL safety boundary

- HIL runner 預設唯讀，只有 `--execute` 加完整確認字串才可能送 goal。
- preflight 與第一個 goal 前都會重讀 map-frame 起點；非有限座標、禁區 frame 不符或
  起點落入禁區一律 fail closed，並保存未送 goal 的 artifact。
- 目前 Reset 後合法起點的 live route／cancel／stop 與 Hero Demo 10-run **尚未完成**；
  不得將 runner 存在解讀為 live HIL 已通過。

## Release supply chain and installation

- release gate 產生 wheel、sdist、matching constraints、CycloneDX SBOM、`SHA256SUMS`
  與 GitHub artifact attestations，並以 fresh constrained environment 驗證雙 CLI entry、
  doctor 與 uninstall lifecycle；published release 不可覆寫。
- repository 目前是 private，正式安裝流程要求有權限的協作者先登入 GitHub CLI，再以
  `gh release download` 取得同一 release 的資產。匿名下載目前不受支援。
- 歷史 `v1.1.4` 只有 wheel／sdist，缺少 matching constraints 與 checksum；不能回溯
  宣稱它通過本版供應鏈閘。

## Evidence limits

- E2 是固定目標的描述性配對再分析（A／B derived、C observed），不是前瞻性三條件消融。
- B4 固定 subset 為 102 份 reports、407／408 waypoint succeeded；歷史約 20 h 摘要
  不能證明精確暴露量或零安全事件。
- 使用者效率對照、非 Ackermann 物理 PoC、獨立事件觀察、第二維護者演練，以及合法
  起點 HIL／10-run 仍是外部證據閘；本版不把這些未完成項目包裝成成功。
- M6 `perceive → decide → rehearse → act → feedback` 常駐迴圈移至 post-v2（候選 v3）
  研究方向，沒有在 v2.0.0 中實作。

## Upgrade checklist

1. 閱讀新的批准語意；不要預期既有 P2／host remember 能延續。
2. 執行 `JenAI data status`，先以 `JenAI data harden --dry-run` 檢視 legacy 權限，
   確認共享需求後才選擇是否執行 harden。
3. 依 [QUICKSTART](../QUICKSTART.md) 使用同版 wheel、constraints 與 checksum 安裝，
   再跑 `JenAI version` 與 `JenAI doctor`。
4. 接 Isaac／Nav2 前先跑唯讀 HIL preflight；Reset／Play 到合法起點並安排 stop 操作者後，
   才能執行 live 驗收。
