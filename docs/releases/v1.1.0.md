# JenAI v1.1.0 — Bounded autonomy and verified ROS2 actions

JenAI v1.1.0 extends the supervised robot decision boundary with bounded known-location exploration and an end-to-end ROS2 developer workflow. The agent still selects and invokes existing capabilities; low-level motion, approval, navigation gateways, Twin Gate validation, and runtime limits remain outside the LLM.

## Highlights

- Added `/explore` for low-repeat random patrol over eligible saved locations, bounded by duration, navigation-goal count, consecutive failures, optional tags, and an optional reproducible seed.
- Enabled natural-language patrol requests to invoke the same bounded exploration capability through an approval card, including compatibility with OpenAI-compatible local models that select the tool directly.
- Added a focused ROS Developer agent that discovers live topics and schemas, performs one approved bounded action, auto-stops, then verifies observed odometry feedback without repeating actuation on missing feedback.
- Preserved the decision boundary: the Supervisor can reach the bounded exploration API but cannot call unrestricted route execution directly.
- Corrected Twin Gate outcome handling so `refer` and `block` remain explicit outcomes instead of being mislabeled as generic failures.
- Hardened evaluation and experiment tooling for paired ablation analysis, isolated ROS2 agent tests, and reproducible benchmark metadata.

## Validation

- 503 automated tests passed locally.
- Ruff checks passed for the changed agent routing and exploration files.
- Live TUI + Nav2/Isaac Sim natural-language test reached 2/2 randomly selected tagged locations and stopped at the configured goal limit.
