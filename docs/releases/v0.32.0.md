# JenAI v0.32.0 - 安全邊界補強

**讓既有安全承諾在 Ackermann 控制、WebUI 併發與 timeout 路徑上確實成立，並補齊長時間運行與 ROS2 scaffold 的可靠性邊界。**

## 安全鏈修正
- Ackermann `AckermannDrive` / `AckermannDriveStamped` 的 `speed` 與 `steering_angle` 現在會套用 `[vehicle] max_linear / max_angular`，直接發布 `/ackermann_cmd` 不再繞過執行期硬限速
- WebUI robot confirmation 加入單一動作互斥；已有 drive/navigation 執行時拒絕重疊動作，且保留未使用的 confirmation 供稍後重試
- shell tool 與 `colcon build` 改以獨立 process group 執行；timeout 會先 TERM、必要時 KILL 整個子程序樹，不再留下背景程序繼續執行

## 可靠性修正
- AuditStore 在 transaction commit/rollback 後明確關閉 SQLite connection，消除長時間運行時的 descriptor 累積與 `ResourceWarning`
- ROS2 package scaffold 無條件保留 `rclpy` dependency，即使模型只回傳其他合法 dependency，生成的 `package.xml` 仍維持完整

## 驗證
- Ruff 全綠
- Python 3.12 / 3.13 / 3.14 完整測試全綠
- Coverage 75%；原 SQLite unclosed-database warnings 已消失
- PR CI `test` 與 `build` jobs 全綠
