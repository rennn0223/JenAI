# JenAI v0.25.1 - NVIDIA NIM credential hotfix

## Fixed

- Restored LiteLLM as a declared dependency for existing provider workflows and direct LiteLLM usage inside the JenAI environment.
- The setup wizard now distinguishes an environment-variable name from an accidentally pasted API key. A pasted key is moved into the adjacent `.env` file with mode `0600`; `config.toml` stores only the variable name.
- Config validation rejects literal API keys in `api_key_env`, and validation errors no longer echo the rejected secret.
- A symlinked `.venv` is ignored, allowing the environment to live outside macOS Documents sync when Python 3.13 editable-install `.pth` files are marked hidden.

## Verification

- NVIDIA NIM `/v1/models` returned HTTP 200 and exposed the configured model.
- `401 passed`; lint, build, and wheel smoke-test gates passed.
