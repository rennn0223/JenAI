# JenAI v0.34.2 - 自然語言路徑兩修:goal-only 導航 + agent 位姿

**agent 代測「不用 slash、純自然語言」的產出:修掉 NL 路徑上兩個確定的 bug,並修正 v0.34.1 對「模型迴圈」的錯誤診斷。**

## 修正

- **route 解析器支援 destination-only**:`去X`、`到X`、`前往X`、`Go to X`、`navigate to X` 現在直接解析(regex 快路,不需 provider);LLM fallback 原本「prompt 說缺起點就填空字串、程式卻把空起點整個否決」的自相矛盾一併修掉——只有目的地時一律從機器人當前位置出發(與 TEST.md 既載規格一致)。`/run` agent 內部第一次嘗試就是這句式,修完 live 實測解析即過
- **`ros_state` 新增 `pose` 欄位**:以 latched QoS(RELIABLE+TRANSIENT_LOCAL)讀 `/amcl_pose`——與 v0.33.2 的 bridge 修復同語意。載具把 odom 改名時(Carter=`/chassis/odom`)agent 不再回答「無法得知機器人座標」;`topic_echo` adapter 增加 `latched` 參數
- 工具說明同步:ros_state 明示「導航只需目的地,拿不到位姿不准反問人類出發點」

## 重要更正(v0.34.1 notes 的錯誤診斷)

「qwen3.6:35b 模型迴圈」**是誤判**。audit 證據:resume 轉 running 到 blocked 僅 **28ms**——模型根本沒被呼叫、車沒動;`state.approve()` 已執行,但 SDK(openai-agents 0.17.7)原樣重吐同一 interruption,迴圈偵測器把它當成模型迴圈擋下。真兇在 **approval 對帳**(嫌疑:handoff 後 tool namespace 的 key 解析),已列 TEST.md known issue 3、最高優先。影響範圍:自然語言/`/run` 的所有動作路徑;slash 直接指令不受影響。
