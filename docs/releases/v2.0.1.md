# JenAI v2.0.1

**v2.0.1 是發行工作流的 patch 修正；產品執行行為、CLI、設定 schema、ROS2／Nav2
決策與安全語意均與 v2.0.0 相同。依 `docs/operations/VERSIONING.md`，本次沒有 public surface
破壞或新功能，因此不升 minor／major。**

## v2.0.0 發布事故與不可變歷史

- 2026-07-19 的 manual release run 已建立 annotated tag `v2.0.0`，指向
  `1800e1792d979603be819db1e1891a1b2dc148f1`。
- 該 run 在 `Generate signed build provenance` 失敗；GitHub 回覆 user-owned private
  repository 不提供 attestation persistence，必須改為 public 才可使用此功能。
- `v2.0.0` **沒有建立 GitHub Release，也沒有發布任何 assets**。現有 tag 不刪除、不移動、
  不重用；原 [v2.0.0 notes](v2.0.0.md) 保留作為當時的歷史候選說明，不能當作已發布證據。

## Private/public 發行修正

- 兩個 `actions/attest` steps 只在 repository visibility 為 public 時執行；public path
  必須同時取得 build-provenance 與 SBOM-attestation bundles，缺任一即 fail closed。
- private／其他 non-public path 明確記錄 checksum-only integrity mode，不建立
  `.sigstore.json` bundles，也不宣稱 GitHub provenance。
- 新版 workflow 會為 private release 產生 CycloneDX SBOM 與 `SHA256SUMS`；只有這些 assets 實際出現在該 GitHub Release 時才視為已發布，且 private path 不會產生或宣稱 GitHub artifact attestations。
- 兩種模式都使用同一份明確允許清單來產生 `SHA256SUMS` 與上傳 assets；`uv build`
  產生的 `dist/.gitignore` 或其他非允許檔案不會進入 manifest／release。Private release
  發布 wheel、sdist、matching constraints、CycloneDX SBOM 與 checksum；public release
  另把兩份 attestation bundles 納入同一份清單。
- draft recovery 會在覆寫前後比對遠端 asset 名稱；出現舊版或清單外資產時 fail closed，
  不把混合版本的 draft 發布出去。
- 可執行 supply-chain tests 分別跑 private skip、public required、public 缺 bundle 失敗，
  防止只靠 YAML 字串比對造成回歸。

## 安裝與驗證

已授權 private repository 協作者仍以 `gh release download` 取得同版 wheel、constraints
與 `SHA256SUMS`，先驗 checksum 再安裝。`gh attestation verify` 只適用 repository 在該版
發布時為 public、且 release assets 實際附有 `.sigstore.json` bundles 的版本；private
release 不得執行或宣稱 attestation 驗證已通過。
