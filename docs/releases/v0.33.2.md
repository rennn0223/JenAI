# JenAI v0.33.2 - 靜止載具讀不到位姿的 bridge 修復

**首次在 Isaac Carter 場景實測(sim-first 驗證第一天)踩出的真 bug:車靜止時 `/loc add here` 回報 "No pose received"。**

## 修正

- **`get_pose` 對 `/amcl_pose` 改用 `RELIABLE + TRANSIENT_LOCAL` 訂閱**(與 Nav2 AMCL 的發佈 QoS 匹配):AMCL 鎖存最後一筆位姿、只在更新時重發,原本的 `SENSOR_DATA`(VOLATILE)訂閱收不到鎖存樣本——**車不動就永遠等不到位姿**,再 fallback 到 `/odom` 時,Carter 這類把 odom 發在別名(`/chassis/odom`)的載具就整個失敗。`/odom` 維持 `SENSOR_DATA`(連續串流,語意正確)
- Wheeltec 車先前不受影響純屬僥倖(直接發 `/odom` 蓋住了 AMCL 的問題);此修復讓 `/loc add here`、`/route` 起點解析、Twin Gate 的 G3/G4 位姿取樣在靜止狀態一律可用

## 驗證

- 對活的 Isaac Carter 場景(車靜止)以 bridge stdio 實測:`{"op":"pose"}` 立即回傳 `/amcl_pose` 的 map 座標 ✅(修復前同情境 timeout)
- `env -u PYTHONPATH uv run pytest` 全綠;ruff 全綠;PR CI `test` + `build` 全綠
