# JenAI v1.1.1

This patch release freezes the runtime reliability fixes validated after the
v1.1.0 Isaac Sim acceptance run.

## Fixed

- Bounded ROS2 drive now publishes motion and zero-stop pulses from the same
  rclpy publisher. This removes the second ROS CLI process startup delay that
  made short E3 mock motions last roughly twice as long as requested.
- Planning rejects empty plans and normalizes model-authored initial step
  states to `pending`; only execution may advance lifecycle state.
- ROS schema summaries fall back to deterministic field parsing after an
  eight-second model timeout instead of blocking the TUI queue.
- Twin Gate captures synchronous initial and final poses so an already-near
  Nav2 goal cannot be misclassified solely because the background sampler had
  not emitted yet.
- The bounded B4 driver uses the current four-corner location set, records the
  declared route, distinguishes full and partial patrol results, retains
  single-instance locking, and sends `/stop` on exit.
- Navigation-agent instructions now require a valid route preview followed by
  `route_execute_tool` in the same run. Approval remains enforced by the
  framework, while auto mode no longer depends on a model-authored prose
  confirmation.

## Validation

- Python 3.12-3.14 CI matrix, lint, package build, and wheel smoke test.
- Isolated ROS_DOMAIN_ID=42 measurement: 0.10 m/s for one second produced
  0.10003 m in the deterministic mock UGV after the publisher fix.
- The frozen eight-case E3 batch passed 7/8 and respected the no-repeat
  actuation boundary in 8/8. The sole failure was a nonexistent-tool model
  call before actuation; a separately declared three-motion run passed 3/3.
- A one-lap four-corner Isaac Sim/Nav2 preflight reached 3/4 points in 720 s.
  The final goal was referred after Twin Gate observed a G5 Nav2 abort, so the
  execution-side goal was not sent. This supplements rather than replaces the
  earlier approximately 20-hour B4 result.
- Isaac Sim/Nav2 natural-language and bounded B4 results are documented as
  simulation evidence and do not claim physical-vehicle safety validation.
