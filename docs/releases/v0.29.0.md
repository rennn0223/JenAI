# JenAI v0.29.0 - Review 修復:堵住殘餘的 fail-open 與 TUI 崩潰面

**v0.28.0 safety hardening 的收尾:high-effort code review 找到的 7 個 findings 全數修復——非有限孿生 pose 不再被當成有效的禁區檢查證據,使用者輸入不再能讓 TUI 崩潰。**

## Safety

- Twin Gate G3 取樣只把**有限座標**算成有效樣本:NaN/inf 的 pose 永遠不會落在任何禁區(比較恆為 False),先前會讓軌跡「看起來檢查過了」而放行;現在一律落入 inconclusive 路徑(`refer`)。
- `/api/stop` 維持「先停車、再讀 body」,但回應後會在 2 秒逾時內排空小型 request body——瀏覽器不會再因 TCP RST 把成功的急停顯示成 network error,謊報長度的 client 也無法佔住 handler thread。
- WebUI 對「合法 JSON 但非 object」的 body(如 `[1,2,3]`)回 400,不再靜默視為空指令。

## Reliability

- 使用者輸入與例外訊息進時間軸前一律 escape:貼上含 `[/]` 的文字(ROS log、程式碼)先前會在 compositor 拋 `MarkupError` 讓整個 TUI 崩潰。
- 寒暄捷徑補上 JenAI persona system prompt(免工具模式、不得聲稱操作過機器人),並把該輪對話寫入 session 記憶——agent 下一輪不再失憶。
- `/loc add here` 遇到 NaN/inf 的即時 pose 時,給出「檢查 localization」的明確診斷,而非裸露的 pydantic ValidationError;不寫入任何地點。

## Verification

- `429 passed`(新增 5 個回歸測試,含「移除修復後測試必失敗」驗證)。
- `ruff check src tests` 通過。
- 實測:謊報 Content-Length 的 `/api/stop` 立即回應且 handler thread 正常釋放。
