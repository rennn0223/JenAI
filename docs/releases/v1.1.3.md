# JenAI v1.1.3

This patch hardens natural-language navigation follow-up turns and adds product-readiness
governance without changing the approval, stop, limit, or NavigationGateway safety semantics.

## Fixed

- Mirrored the bounded navigation tool chain on the supervisor so compatible local models that
  omit a Navigation handoff no longer fail with `Tool not found` on a follow-up request.
- Closed failed and Esc-interrupted file-session turns with an honest assistant marker, preventing
  dangling user turns from slowing or confusing the next request.
- Preserved route execution approval and the existing Twin/NavigationGateway checks; raw ROS
  drive and publish tools remain absent from the supervisor.

## Product readiness

- Added a six-role acceptance matrix for engineering, product, operations, research, sales, and
  buyer concerns, with `PASS`/`PARTIAL`/`OPEN` evidence gates.
- Added installation, support, security, rollback, adoption-cost, support-matrix, and second-vehicle
  PoC boundaries; Apache-2.0 remains the open-source core license.
- Added a pseudonymous manual/Slash/natural-language usability-study recorder that preserves
  failures and excludes ambiguous repeated pairs from paired speed ratios.
- Added Dependabot plus locked dependency audit and CycloneDX SBOM workflows.

## Evidence and limits

- Recorded the 2026-07-18 live TUI failure and fix: follow-up lookup/preview reached the approval
  card after the patch, rejection caused no movement, and Esc session repair was verified.
- The same run preserved G3 blocks, one pre-fix tool error, one 192-second generation, and zero
  successful navigation trials; see `docs/validation/TUI_LIVE_ACCEPTANCE_2026-07-18.md`.
- This release does not claim Hero demo 9/10, quantified efficiency, fresh-user onboarding,
  Twin domain isolation, physical safety, or cross-vehicle physical generalization.
