# JenAI v2.1.0

**v2.1.0 makes routine robot inspection substantially more responsive while preserving
the same approval and execution boundaries for decisions and motion. It also completes
the accepted terminal UI refresh.**

## Deterministic read-only fast path

- Explicit natural-language requests that only inspect pose, LaserScan, and Nav2
  readiness now call the same recorded read-only ROS tool directly, without waiting for
  the local LLM.
- Mixed requests such as “check status, then return to dock” are excluded from the fast
  path and retain the full model, approval, NavigationGateway, and result-verification
  flow. The router cannot silently discard the requested action.
- Independent pose, odometry, and scan snapshots run concurrently, so one unavailable
  optional topic no longer serially delays the others.
- A partial ROS topic graph is retried once during cold startup, preventing transient
  discovery from being reported as a persistent Nav2 failure.
- Agent sessions are bounded by both item count and serialized bytes, limiting prompt
  growth from large ROS payloads while trimming only at user-turn boundaries.

## Natural-language and terminal polish

- Saved-location lookup accepts harmless model-supplied English articles such as
  `the dock`, while destructive location operations still require an exact saved name.
- The Claude Code-inspired terminal welcome screen uses the accepted animated
  dachshund, clearer two-column hierarchy, recent activity, and responsive compact
  status lines for narrow terminals.
- Read-only state runs keep the same timeline and audit records as agent tool calls and
  return deterministic measured summaries rather than an extra model-generated claim.

## Candidate validation

- The local suite passes 675/675 tests; Ruff and `git diff --check` pass.
- Wheel and sdist build successfully, and an isolated install reports `JenAI 2.1.0`.
- A supervised v2.1.0 Isaac/Nav2 TUI check completed the read-only natural-language state
  path in 5.120 seconds with all five readiness checks passing and no movement command.
  This is a one-shot engineering observation,
  not a latency distribution or replacement for the E4 LLM benchmark.
- A natural-language dock request still reached an approval card; rejecting it with Esc
  produced no movement. Historical v2.0.2 FullScan/Hero evidence remains historical and
  is not relabeled as v2.1.0 HIL evidence.
- GitHub CI, merged revision, tag, release assets, SBOM, and checksums are authoritative
  only after the corresponding remote workflows and published release complete.
