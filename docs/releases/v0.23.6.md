**`JenAI web` 列印所有開法** —— 應使用者回饋:「有 token 就很安全,可以在 JenAI web 的時候直接把好幾個可開方式都列印嗎?」可以,而且守住一條誠實原則:**只印真的打得開的網址**。

## 改了什麼
- **綁 127.0.0.1(預設)時列印**:本機網址(含 token)、SSH 轉發一行指令(`ssh -L 8760:127.0.0.1:8760 …` 後開同一網址)、以及「想區網直連就 `--host 0.0.0.0`」的指引——不印打不開的區網網址
- **綁 0.0.0.0 時列印**:本機網址 + **各網路介面的區網網址**(新 helper `lan_addresses()`,UDP connect 探測不發包 + hostname 解析,排除 loopback)+ SSH 轉發仍通
- 固定書籤提示:token 每次重啟會換,`--token` 可固定
- 輸出組裝抽成純函數 `_web_access_lines()`,可單測

## SSH 打不開的三個常見原因(啟動輸出現在直接教你解)
1. WebUI 預設只綁 Jetson 的 loopback → 筆電直連 `<jetson-ip>:8760` 必然不通 → 用 SSH 轉發或 `--host 0.0.0.0`
2. token 每次重啟換新 → 舊書籤 401 → `--token` 固定
3. 上一個 web 沒關佔住 port → `ss -ltnp | grep 8760` 清掉或 `--port` 換

## 驗證
- 新增 `lan_addresses` 不含 loopback 測試、`_web_access_lines` 兩種綁定的誠實性測試(loopback 不得出現區網網址)——**380 total** 全綠;ruff 全綠
- 實際列印兩種綁定的輸出人工核對
