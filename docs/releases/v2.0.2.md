# JenAI v2.0.2

**v2.0.2 strengthens Isaac Sim / Nav2 evidence quality and prevents a same-domain
Twin configuration from sending a navigation target twice. It does not change the
approved route format, saved-location format, or low-level ROS 2 control APIs.**

## Navigation and HIL hardening

- Nav2 cancellation now preserves whether the action server acknowledged the
  cancellation; local task cancellation alone is never reported as a confirmed stop.
- Stale terminal results from an earlier goal cannot complete a newer navigation task.
- FullScan acceptance now checks sample count, angular coverage, bin count, geometry,
  range bounds, scan timing, frame stability, advancing timestamps, all-infinite sample
  ratio, and finite-bin coverage before a route may execute.
- HIL progress evidence is bounded while preserving terminal state, and route payloads
  use one canonical, fail-closed normalizer for approval and execution.
- If the optional Twin and the target share `ROS_DOMAIN_ID`, JenAI explicitly skips the
  non-isolated rehearsal and sends one target goal. A genuinely isolated Twin domain
  continues to require its pass verdict before target execution.

## Candidate validation

- The clean `cc6d217` candidate passed the formal Isaac/Nav2 route, return-to-Dock,
  cancel-acknowledgement, and zero-drift checks with the strengthened FullScan gate.
- A 10-leg alternating `map_left_down` / `dock` Isaac demonstration completed 10/10;
  the retained first attempt failed closed before movement during a transient pose-feed
  outage, followed by a successful recovery preflight.
- Final full-suite, natural-language single-goal, package, pull-request, and release
  checks are release gates and will be recorded here before publication.
