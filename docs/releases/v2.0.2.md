# JenAI v2.0.2

**v2.0.2 strengthens Isaac Sim / Nav2 evidence quality and prevents a same-domain
Twin configuration from sending a navigation target twice. It does not change the
approved route format, saved-location format, or low-level ROS 2 control APIs.**

## Navigation and HIL hardening

- Nav2 cancellation now preserves whether the action server acknowledged the
  cancellation; local task cancellation alone is never reported as a confirmed stop.
- Stale terminal results from an earlier goal cannot complete a newer navigation task.
- FullScan acceptance now checks sample count, angular coverage, bin count, geometry,
  range bounds, scan timing, frame stability, advancing timestamps, all-infinite sample
  ratio, and finite-bin coverage before a route may execute.
- HIL progress evidence is bounded while preserving terminal state, and route payloads
  use one canonical, fail-closed normalizer for approval and execution.
- If the optional Twin and the target share `ROS_DOMAIN_ID`, JenAI explicitly skips the
  non-isolated rehearsal and sends one target goal. A genuinely isolated Twin domain
  continues to require its pass verdict before target execution.

## Candidate validation

- The clean `d942130` candidate passed the final Isaac/Nav2 run: 10/10 FullScan
  samples with all metadata gates, two routes with zero recoveries, acknowledged Nav2
  cancellation, and 0.0000 m stop drift. Twin isolation remained an explicit same-domain
  skip; the artifact does not claim a Twin verdict or physical safety.
- On clean `cc6d217`, the retained first Hero attempt failed closed with zero goals during
  a transient pose-feed outage. Recovery preflight passed, then an alternating
  `map_left_down` / `dock` sequence completed 10/10 deterministic Nav2 route legs.
  This is not ten natural-language trials or ten complete three-minute demos.
- One supervised natural-language TUI request returned to Dock and increased the Nav2
  goal count by exactly one, with no invalid JSON. It ran from a dirty patch later
  committed as `d942130`, so it is supplementary interaction evidence, not a clean-run
  success rate.
- The local strict suite passed 652/652; Ruff, wheel/sdist build, both CLI entry points,
  and an isolated wheel smoke test passed. PR #108 passed Python 3.12/3.13/3.14, build,
  and audit/SBOM checks. Publication is valid only when the merged revision, tag,
  published assets, and downloaded-asset verification are observed on GitHub; the
  GitHub Release is authoritative for that post-publication evidence.
