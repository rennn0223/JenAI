# JenAI v0.30.0 - Runtime reliability, auditability, and leaner deployment

**這個版本把 code review 找到的跨程序可靠性缺口補齊，加入可跨重啟查核的 metadata audit trail，並讓機器人端部署更精簡。**

## Reliability

- WebUI map 不再輸出 NaN/inf：localization pose 非有限時明確回報 invalid，不把壞資料送進 JSON 或誤當有效位置。
- SessionState 以同路徑共享鎖與 OS advisory lock 保護，跨 instance／process 的 add、pop、clear 不再互相覆寫。
- WebUI 所有 client 共用有界狀態快取：doctor 30 秒、ROS graph 2 秒，同時刷新只執行一次 probe。

## Auditability

- 新增有界 SQLite audit store，跨重啟保留 run、approval、tool 與 Twin Gate verdict 事件。
- audit 最多保存 10,000 筆，且刻意不落盤 prompt、raw action 或完整 payload。
- TUI、CLI、WebUI、daemon 與 MCP 共用相同 audit 邊界；audit 寫入失敗不阻塞安全動作。

## Safety engineering

- Bridge watchdog 與 navigation result／active-state 抽成 stdlib-only sibling modules，可在沒有 rclpy 的 CI 直接單元測試。
- Safety-chain coverage gate 納入 bridge 純邏輯模組，降低 ROS callback 接線以外的回歸盲區。

## Dependencies and compatibility

- LiteLLM 改為遠端 OpenAI-compatible gateway：機器人端移除未使用的 LiteLLM 與 20 個傳遞依賴，lockfile 從 87 降為 66 個套件。
- CI 明確覆蓋 Python 3.12、3.13、3.14，並以契約測試防止版本與 living docs 漂移。
- README、架構、測試、威脅模型、roadmap 與目錄說明同步到實際行為。
