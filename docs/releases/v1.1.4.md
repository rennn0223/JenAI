# JenAI v1.1.4

**這一版把 Isaac Sim 驗收從人工筆記提升為可稽核、自願啟動的 HIL 流程，
並在不改變既有 UI 視覺與安全語意的前提下拆分高風險大型模組。**

## Isaac HIL acceptance

- 新增 `scripts/isaac_hil_acceptance.py`：預設只做唯讀 preflight；只有
  `--execute` 加完整 `I-CONFIRM-ISAAC-SIM-MAY-MOVE` 才可送 Nav2 goal。
- 新增人工 `workflow_dispatch` 的 `isaac-hil.yml`。一般 push、PR 與排程永遠
  不會動車；self-hosted runner 會驗 route、取消、hard stop 與可選 Twin verdict，
  並在成功或失敗後上傳 JSON artifact。
- HIL preflight 對 ROS graph discovery 做最多三次有限重試，artifact 保留每次
  transient failure，不用最後一次成功覆蓋先前異常。
- 同 domain 純模擬模式不會被包裝成 Twin isolation：未要求 Twin 時明確 `skip`；
  加 `--require-twin` 時 fail closed 且不送 target goal。
- artifact 保存設定 SHA-256、JenAI 版本、ROS domain、progress、GateReport 與停止後
  drift；不保存金鑰、prompt、主機名或設定絕對路徑，且預設拒絕覆寫舊證據。

## Maintainability

- `NavigationGateway` 新增只讀的 structured GateReport observer，HIL 可保存
  `pass/block/refer`，不影響原 gate decision 或 audit。
- 將 Slash catalog、standalone greeting 判斷與原 CSS 原樣移至
  `tui/catalog.py`；本版沒有套用待確認的 UI/UX 樣稿。
- 將 `/loc` persistence 與 pose/GPS 處理拆成 `LocationCommandsMixin`，
  `robot_commands.py` 由 1,101 行降至 812 行。
- 將 bridge wire op mapping 拆成可在 system Python 與 package test 共用的
  stdlib-only `bridge/_protocol.py`；`ros_bridge.py` 由 935 行降至 875 行。

## Validation

- 本機完整 `532/532` pytest 通過，完整 Ruff、lock check、wheel/sdist build 通過。
- 66 項 TUI/Help、84 項 TUI/location、30 項 bridge protocol/safety/client
  針對性回歸通過；system-Python sibling import 與 sidecar 語法編譯通過。
- 2026-07-18 真 ROS/Isaac graph 唯讀 preflight 通過 required nav checks；
  首輪曾短暫漏 `/amcl_pose`，第二輪恢復，兩次結果都保存在本機 ignored artifact。
- 該 preflight 同時保留 `twin_isolation=fail`，因目前純模擬設定刻意讓 target 與
  Twin 都使用 domain 0。

## Evidence limits

- 本版尚未產生 self-hosted **live** HIL artifact，也尚未完成 Hero Demo 10-run。
- 不聲稱 Twin domain isolation、實體安全、跨載具物理泛化、量化效率提升或
  fresh-user onboarding 已完成。
- Claude Code CLI 風格 UI 仍只有獨立樣稿；依使用者要求，在樣稿確認前不改現有介面。
