# JenAI v2.2.0

**v2.2.0 turns saved robot actions into site-bound, evidence-aware product contracts.
It keeps the LLM responsible for high-level decisions while making execution boundaries
and final reports deterministic, inspectable, and honest.**

## Site-bound navigation

- An active Site Profile now binds the map SHA-256 identity, saved locations file,
  validated routes, Dock location, and validation evidence.
- High-level navigation fails closed when the Site Profile is inactive or its map
  identity does not match the live OccupancyGrid.
- Product navigation uses Nav2 through one NavigationGateway. Direct odometry driving
  remains an internal bridge bring-up mechanism and is not exposed as a product route.
- Rotated OccupancyGrid origins are transformed correctly before map-cell checks.

## Honest task outcomes

- A successful Nav2 arrival at Dock is reported as `arrived_unverified`, because Isaac
  Sim currently provides no charging-state feedback.
- Patrol and exploration runs that reach a waypoint but miss required photo evidence
  are reported as `partial`, rather than silently promoted to success.
- Failed tool calls remain visible even when the surrounding agent lifecycle completed.
  Runs without an explicit verified outcome default to `partial`, never implicit success.
- Natural-language, Slash, MCP, daemon, and direct TUI routes share the same gateway,
  outcome vocabulary, audit trail, and approval boundary.

## Capability contracts and documentation

- One capability catalog now defines English and Traditional Chinese descriptions,
  supported platforms, limitations, risk, approval policy, and runtime behavior.
- UGV and quadruped profiles advertise only capabilities they can actually execute.
- The new NVIDIA-inspired documentation site consolidates Quick Start, architecture,
  Site Profile setup, task outcomes, release gates, and known limitations.

## Candidate validation

- The complete local pytest suite, Ruff formatting/lint, mypy strict, wheel/sdist build,
  and isolated wheel smoke test pass.
- The 2026-07-24 Isaac/Nav2 engineering run verified Site Profile map identity, LaserScan
  quality, two route arrivals, acknowledged cancellation, and software halt. The final
  endpoint deviations were 0.043 m for a route and 0.038 m for Dock.
- A post-replay read-only preflight passed with 10/10 LaserScan samples, 50.2% finite-bin
  coverage, a free localized start, and feasible Nav2 plans for the requested routes.
- These are simulation engineering results, not physical-vehicle or charging proof.
  Configured domain separation does not by itself prove live cross-domain isolation.

GitHub CI, merged revision, release assets, SBOM, checksums, and provenance claims become
authoritative only after the corresponding remote workflows and published release pass.
